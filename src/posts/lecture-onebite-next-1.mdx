# 소개

Next.js = React.js 전용의 웹 개발 프레임워크

현재 많은 사랑을 받고 있는 기술!

### 프레임워크와 라이브러리의 차이

- 기능 구현의 주도권이 누구에게 있는가
- 프레임워크 = 프레임워크, 라이브러리 = 개발자

# Next.js의 사전렌더링

### CSR (Client Side Redering)

클라이언트에서 직접 화면을 렌더링

1. 서버에서 빈 index.html을 받아서 일단 빈 화면을 렌더링
2. 그 후 JS Bundle을 받아 브라우저에서 실행하고 화면에 렌더링

초기 접속에 서버로부터 미리 다 받아서 준비해놨기 때문에 그 후 페이지 이동이 빠르고 쾌적하다. 단, 초기 접속이 느리다는 단점이 있다 (FCP가 늦다)

### FCP (First Contentful Paint)

요청 후 화면에 나타나기까지 걸리는 시간

### 사전렌더링이란?

브라우저의 요청에 사전에 서버에서 렌더링이 완료된 HTML을 응답하는 방식

1. 초기접속에서 서버에서 렌더링이 완료된 HTML을 받아서 화면에 바로 렌더링하고 (FCP가 CSR 보다 빠르다)
2. 그 후 JS Bundle을 받아 수화(Hydration)과정을 거치고 상호작용이 가능하게 한다
3. 추후 접속은 CSR 방식으로 처리된다

그래서 Next.js의 사전렌더링은 CSR 방식의 단점이었던 FCP가 느리다는 단점은 해결하면서 CSR 방식의 장점인 빠른 페이지 이동은 가져가는 방식이라고 할 수 있다
